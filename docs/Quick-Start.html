Quick Start

Requirements

To get started with Montage, you will need the following:

  • (optional) An account on GitHub.
  • A GIT client, and public SSH key. For details on installing GIT and creating your key, see the setup guides for Mac and Windows.
  • A local web server to serve the Montage framework and applications. Montage applications can only be served from an HTTP address, not from a local file URL.
  • A recent stable release of Chrome or Safari.

Quick setup

If you’re already familiar with using Git, are familiar with GitHub, and able to configure a local web server (preferably Apache), then this section will get you started with Montage right away.

  1. In a terminal window, create a new projects folder (eg. ~/Projects). 
    $ mkdir Projects
  2. Clone Montage into ~/Projects. 
    $ cd Projects
$ git clone git@github.com:montagejs/montage.git

    This will will clone the Montage source into ~/Projects/montage.
  3. Configure your local web server to serve the ~/Projects folder over HTTP. For a suggested Apache httpd.conf file configuration, see apache-montage.conf.
  4. Verify your setup by browsing to http://localhost:8081/montage/samples/photofx/index.html.

If you successfully ran the PhotoFX sample, check out the Hello Montage tutorial.

Complete setup steps

OS X: Using Montage with the default Apache web server

You can use the default Apache web server with OS X to test your Montage applications. To enable the web server, open the Sharing pane in System Preferences, and enable the Web Sharing option.

  1. Next, in a Terminal window change directory to your Apache web root folder, ~/Sites by default.
  2. Clone the Montage repository into the ~/Sites folder. Note: Mac OS X Lion users will need to manually create the Sites folder in their user folder (/Users/_userName_/Sites).
  3. Test your environment by pointing your browser at the PhotoFX sample application, replacing username with your actual username.

Local web server configuration tips

In general, it’s recommended that 1. you create a projects folder, 2. clone the Montage repository into that folder, and 3. configure your web server’s document root to the point to the Montage distribution folder (montage), or create a virtual directory that points to the same location.

It’s also recommended that you disable caching on your web server during development; this ensures that any changes made to your project’s files on disk are reflected in the browser when you reload the project in your web browser. If you are running an Apache web server, you can also apply the recommended httpd.conf settings configuration to your projects folder. The "no-cache" HTTP header setting is important as it lets you iterate on your HTML, CSS and JavaScript files without having to worry about cached versions.
