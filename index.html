What is MontageJS?

MontageJS is a modern, full stack HTML5 framework designed to create single-page applications—fast! MontageJS builds on the strength of Cocoa and iOS to provide HTML5 with the capability to build native-class web applications.

Why use MontageJS?

MontageJS uses time-tested design patterns and software principles, allowing you to easily create a modular architecture for your projects. Key benefits of MontageJS include:

Features

Reusable Components

MontageJS simplifies rich-client development by offering a modular approach to organizing code and components. This allows small teams to write applications faster through maximum reusability and enables larger teams to be more efficient by breaking complex problems into smaller pieces without stepping on each other during the development process

High Performance

MontageJS supports implicit event delegation and a managed draw cycle. This minimizes expensive layout reflows to provide a better user experience, particularly on hardware limited mobile devices.

Functional Reactive Bindings (FRB)

MontageJS supports simple and two-way data binding between components, objects, and collections, allowing you to coordinate the state between views and models, among other entangled objects. FRB go farther and can gracefully bind long property paths and the contents of collections, as well as incrementally update the results of chains of queries including maps, flattened arrays, sums, and averages. FRB makes it easy to ensure consistent state without having to write a lot of “glue” code.

Workflow

When it comes to a smooth workflow, it's important to keep things flexible for multiple team members and reduce the risk of conflicts when contributing code. It's also important that the boring parts can be automated, so you can concentrate on the exciting stuff. With the MontageJS workflow we try to accommodate that.

1. Initialize

To get up to speed fast, minit (the MontageJS Initilizer) will be a great help. It's a command line tool that provides convenient automated tasks during the development phase. Use it to generate all directories and the boilerplate code needed to get your App ready for development.

2. Collaborate

One of the great things about MontageJS is its modular approach where an application is broken into smaller parts (components). This is great since it allows multiple team members work simultaneously on different parts of the application.

Furthermore, each component is divided into HTML, CSS and JS files for a nice developer/designer workflow. Developers can focus on wiring up components and "make it work", while designers polish the UI or add custom styling.

3. Ship

Once it's time to ship, run your application through mop (MontageJS Optimizer). It's a command line tool that minifies (reduces file size) and creates bundles (reduces the number of requests) of your Montage application. This will help improving loading times and performance for a better user experience.

Themes

MontageJS provides the most common components in an abstract form. This lets you build your own custom looking theme that works hand in hand with the framework. On top of that we provide 3 premade themes that should get you up and running quickly.

Digit is a touch-friendly theme optimized for mobile.
Matte contains desktop-optimized UI components.
Native uses the browser's default styles.
